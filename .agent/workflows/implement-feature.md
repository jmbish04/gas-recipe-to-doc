# Workflow: Cloudflare Images Integration for Google Docs

## Objective
Implement a persistence layer for images generated by AI. Ephemeral AI URLs are replaced with permanent assets stored in Cloudflare Images before being inserted into Google Docs.

## Steps
1. **Configure Secrets:**
   - Ensure `CLOUDFLARE_IMAGES_STREAM_TOKEN` is set in Google Apps Script Project Settings > Script Properties.
   - Ensure `CLOUDFLARE_ACCOUNT_ID` is set in Script Properties.
2. **Update Service Logic:**
   - Modify `services/googleDocs.gs` to include the `UrlFetchApp` call to the Cloudflare `/images/v1` endpoint.
   - Refactor image retrieval to pull from the `uploadResult.result.variants[0]` delivery URL.
3. **Safety Checks:**
   - Add zero-division protection to aspect ratio calculations (`originalWidth || 1`).
   - Add `Math.max(height, 1)` to prevent zero-height image errors in Google Docs.
4. **Validation:**
   - Test with a temporary OpenAI DALL-E URL.
   - Verify that the image persists in the Google Doc even after the original URL expires.
